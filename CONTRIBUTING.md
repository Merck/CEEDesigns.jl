Thanks for taking the time to contribute — we appreciate it very much!

We welcome contributions to both regimes of CEEDesigns.jl: the static regime, which fixes one cost-efficient design for a population, and the generative regime, which builds an adaptive sequential design for a single entity. Contributions that extend either regime — new uncertainty measures, distance functionals, value functions, or planners — as well as integrations with the broader Julia machine-learning and decision-making ecosystem are especially welcome.

Please see the [online documentation](https://merck.github.io/CEEDesigns.jl/stable/) for core design principles, the API reference, and worked tutorials. The runnable tutorials under [`tutorials/`](https://github.com/Merck/CEEDesigns.jl/tree/main/tutorials) are the best starting point for understanding how the pieces compose, and the [API documentation](https://merck.github.io/CEEDesigns.jl/stable/api.html) documents the extension points for custom uncertainty measures, distances, and planners.

Contributions to CEEDesigns.jl are welcome in the following forms:

- Modifying the code or documentation via a pull request.
- Reporting bugs or suggesting enhancements in the project's [GitHub Issues](https://github.com/Merck/CEEDesigns.jl/issues). If you propose a feature for future development, we are happy to discuss and take on the implementation.
